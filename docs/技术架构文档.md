## 1. 系统架构概览

### 1.1 整体架构

MultiTrack 采用经典的前后端分离架构：

- **前端**: React + TypeScript + Vite，提供 Web UI
- **后端**: Python FastAPI，提供 RESTful API
- **通信**: HTTP REST API

### 1.2 后端模块结构

- src/backend/api/ — API 路由层
- src/backend/engine/separator/ — Demucs HT 分离引擎
- src/backend/engine/mixer/ — 音频混音引擎
- src/backend/engine/vc/ — 人声变声引擎
- src/backend/utils/ — 音频工具和文件管理
- src/backend/config.py — 全局配置

### 1.3 前端模块结构

- src/frontend/src/pages/ — StepImport, StepSeparate, StepVoiceConvert, StepExport
- src/frontend/src/components/ — SpectrogramViewer 频谱可视化
- src/frontend/src/stores/ — Zustand 状态管理
- src/frontend/src/utils/ — API 客户端和音频播放

## 2. 核心技术：Demucs Hybrid Transformer

### 2.1 技术选型

核心分离引擎: Demucs v4 (Hybrid Transformer Demucs)

| 对比维度 | 旧 STFT 算法(v1-v15) | Demucs HT (v16) |
|----------|----------------------|------------------|
| 方法 | 频谱掩码 + 维纳滤波 | 深度学习混合 Transformer |
| 模型架构 | 无（手工规则） | U-Net + Transformer 注意力 |
| 训练数据 | 无 | 大规模音乐数据集 |
| 分离质量 | BGM 残留，人声串扰 | 相关系数 ~0.04 |
| 频率重叠处理 | 无法分离 | Transformer 频域模式分离 |
| 处理速度 | 快(秒级) | 中等(130s 处理 220s 音频) |

### 2.2 Demucs HT 工作原理

1. **混合架构**: 卷积 U-Net + Transformer 注意力机制
2. **频率-时间双域处理**: 时域保留相位，频域处理幅度
3. **多源输出**: 一次推理输出 4-6 个音轨

### 2.3 为什么 Demucs HT 比旧 STFT 效果好

#### 频率重叠
旧 STFT 无法区分同频段人声和乐器（如人声 200-2000Hz 和小提琴 200-3000Hz），Demucs 通过学习人声频谱模式实现精确分离。

#### 时域连续性
旧 STFT 逐帧处理产生断续音。Demucs U-Net 覆盖 7 秒窗口，输出平滑连续。

#### 相位重建
旧 STFT 使用 Griffin-Lim 估计导致相位失真。Demucs 时域分支直接输出波形保留相位。

#### 验证结果
- 旧版 3s 测试片段输出噪声 -> Demucs 高质量分离
- 220s MP3 -> 19MB 高质量音轨
- 人声+BGM 相关系数 < 0.1
